Estructura de datos

Listado de sub-tareas del ejercicio:
------------------------------------
Resumen del video:
1- Array: Un array es un medio de guardar un conjunto de objetos de la misma clase. Se accede a cada elemento individual del array mediante un número entero denominado índice
2- Lista enlazada: Los datos se unen con punteros. Así la búsqueda se realiza de manera muy rápida. El añadir y quitar también se realiza de forma muy fácil. El problema es que la búsqueda se realiza a partir  del punto de inicio de la lista. 
3- Tablas de hash. Son de las más usadas de entre la estructuras de datos actualmente. Guarda los elementos en parejas clave-valor. Para hallar el indice de cada par se usan las funciones de hash que permiten acceder a los elementos de forma instantánea.
4- Pilas: LIFO, Last In Last Out
5- Colas: FIFO, First In First Out
6- Grafos: Estructura compleja para multitud de propósitos, Se usa para establecer relaciones. Por ejemplo los árboles son grafos. Para algorítmos de búsqueda es muy útil. Por ejemplo los arboles de Merkel, usados en bitcoins.
7- Heaps: Arbol binario completo. Puede ser min heap p max heap. Se impelente con colas de prioridad. Son muy rápidos en obtener mínimos y máximos de forma constante. El precio es que meter o quitar elementos puede tener un coste logaritmos al remover datos.
